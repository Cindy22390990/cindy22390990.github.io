📒 英文錯題本 (English Error Notebook)

一款專為手機與碎片化學習設計的「英文錯題管理系統」。透過分層分類與測驗模式，幫助學習者有系統地整理英文錯題，隨時隨地突破學習盲點。

🚀 線上即時預覽 (Live Demo): 點擊這裡體驗 (GitHub Pages)

📖 專案初衷與學習反思 (My Journey & Reflection)

這個專案的誕生，源自於我在學校大一網頁設計課程中的啟發。在課堂上學習了基礎的 HTML 與 CSS 後，我對程式開發產生了濃厚的興趣，便開始透過網路影片，自主鑽研更深入的 HTML、CSS 與 JavaScript 技術。

當時大一必修英文課需要不斷複習與做筆記，但我卻常常忘記帶實體筆記本。於是，我萌生了開發這款手機版「英文錯題本」的想法，希望能隨時隨地用手機複習。

對於當時的我來說，要從零到有打造一個具有資料庫連線的完整網頁應用程式相對困難，因此在這個過程中，我大量使用了 AI 工具來輔助開發。這次的開發經驗讓我收穫極大：

實踐與自學的價值： 我不僅將課堂所學實際應用到專案中，更深刻體認到學校教的內容是有限的，強大的「自主學習能力」才是持續進步的關鍵。

AI 工具的震撼： 我見識到了 AI 工具的強大，它能作為超級助手，協助我把腦海中原本難以實現的想法，一步步化為實際存在的產品。

我在 AI 時代的定位 (產品思維)： 這也讓我反思，在 AI 寫程式如此盛行的時代，我所扮演的角色是什麼？我認為，我更像是一個充滿產品想法的構想者與把關者。把功能「做出來」已經不再是唯一門檻，我不會只滿足於「有就好」。我會站在使用者的角度去思考：這個介面用起來舒服嗎？操作上流暢嗎？使用者在連線或操作時可能會遇到什麼疑惑（例如處理資料庫載入時的等待體驗）？

這個「英文錯題本」專案，正是在這樣不斷追求良好使用者體驗（UX）的思維下誕生的成果。

✨ 核心功能 (Features)

基於手機友善與碎片化學習的設計理念，本專案具備以下核心功能：

🗂 結構化分類系統
將錯題精準分為「文法 (Grammar)」、「片語 (Collocations)」、「單字 (Vocabulary)」三大類別，幫助使用者快速定位並集中火力攻克弱點。

🧠 雙模式複習機制

📋 快速翻模式： 直接顯示題目、錯誤選項、正解與錯因分析，適合考前或通勤時的快速速讀與查閱。

❓ 測驗模式： 隱藏正解與分析，透過主動回想 (Active Recall) 加深大腦記憶，提供類似 Anki 或 Quizlet 的自我測驗體驗。

📝 詳盡的卡片結構
每張錯題卡片不僅記錄題目與答案，更包含「❌ 錯因分析（為何會錯）」與「🎯 記憶口訣/小技巧」，真正做到從錯誤中學習。

📱 手機端優先設計 (Mobile-First)
採用 Tailwind CSS 打造完美的響應式 UI。版面包含大型點擊區與直覺的排版，完美適配手機單手滑動操作，讓您每次 5-10 分鐘的碎片時間都能高效學習。

☁️ 雲端即時同步
整合 Firebase Firestore，實現錯題資料的雲端持久化儲存。無論切換何種裝置，資料都能即時同步不遺失。

📊 進度追蹤標記
卡片內建「標記已複習」功能按鈕，幫助您輕鬆追蹤每道題目的複習狀態。

🛠 系統架構與使用技術 (Tech Stack)

本專案採用輕量化的前端架構搭配無伺服器 (Serverless) 雲端服務：

前端 (Frontend)

核心語言： HTML5, 原生 JavaScript (Vanilla JS)

樣式框架： Tailwind CSS (透過 CDN 引入，實現快速且一致的 UI 構建)

架構特色： 單一頁面應用 (Single HTML File Structure)

雲端與資料庫 (Backend / BaaS)

資料庫： Firebase Firestore (NoSQL 文件資料庫，使用 onSnapshot 實現毫秒級畫面即時更新)

身分驗證： Firebase Authentication (結合系統權杖與匿名登入機制，確保個人資料隔離與安全性)

部署 (Deployment)

託管平台： GitHub Pages
